# Create a message template

This guide walks you through creating an SMS message template in Subflow. Once saved, a template is available to select when composing broadcasts, responding in conversations, and building workflows.

## Before you start

A few things to have ready before you create the template:

- The text you want the template to contain, or a clear idea of what it should say.
- The contact properties or form fields you want to insert as dynamic fields, if any.
- The tags you want to apply for searchability, or a sense of how you plan to organize templates by tag.

## Create an SMS message template

SMS message templates are text-based and can include personalized patient information. The steps below walk you through creating one.

1. Navigate to **Messaging** in the sidebar. Under **Campaigns**, select **Message Templates**. The **Message Templates** tab opens by default.

2. Click **New Message Template** in the top right corner.

3. Enter a name in the **Title** field. The title helps you identify the template when selecting it later. Choose a name that reflects its purpose, such as "Flu vaccination reminder" or "Day 1 onboarding message."

   > **Tip:** Descriptive titles make templates easier to find when composing a broadcast or building a workflow.

4. Write your message in the **Content** field. The content area has a 1,530-character limit.

   - To personalize the message with patient information, click **Personalize** in the content toolbar. A **Contact Properties** list appears, grouped by source. Select a property (such as **First Name**, **Last Name**, or **Phone Number**) to insert it as a dynamic field. When the message is sent, each dynamic field is replaced with the contact's actual value.
   - To start from an existing template instead of writing from scratch, click **Templates** in the content toolbar and select a template. This inserts its content into the field as a starting point.
   - To attach a file or resource, click **Attach** in the content toolbar.

   > **Note:** The inserted contact properties appear as chips in the message content. The team is finalizing the term to use for these items across the documentation — this guide uses "contact property" to match the label shown in the UI. Update this term once the team reaches a decision.

5. Add tags in the **Tags** field (optional). Tags make the template easier to find when filtering or searching the templates list.

6. Click **Save**. The template appears in your list of message templates and is now available to select in broadcasts, conversations, and workflows.
